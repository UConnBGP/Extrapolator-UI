Policy Descriptions

    What_If_Analysis apply the analysis policy on each announcement to determine if an attack causes the conflict. Current the policies implemented are:

    • Time-Based-Heuristic

      • 1. Simple-Time-Based-Heuristic
        • This policy observes the age of an announcement and factors that into the decision about its validity. Route Hijacks tend to be short lived, so long-standing announcements with an invalid ROA are more likely to be valid routes with improperly configured RPKI than actual attacks. This policy will, for example, output “Whitelist” if a year-old announcement has an invalid ROA and the parameter for “Whitelist” is 10 months.
        • Output: analysis decision from 4 possible decisions (0-Enforce, 1-Deprefer, 2-Pass, 3-Whitelist)
        • Input:
          • time record for this announcement in epoch format (integer t),
          • wherether invalid by asn, length or both (boolean invalid_length, boolean invalid_asn),
          • 3 parameters configurable for INVALID_LENGTH (str L1, str L2, str L3),
          • 3 parameters configurable for INVALID_ASN (str A1, str A2, str A3),
          • 3 parameters configurable for INVALID_BOTH (str B1, str B2, str B3).
            • Example: to set L1 to 2 weeks plus 1 hour, L1=“1w:1h” or “14d:1h”
            • Support units are: s-second, m-minute, h-hour, d-day, w-week
            • month and year is not supported due to inaccuracy
        • Each parameter is in the format of the length of time, and is treated as the boundary between the decisions in the timeline.

    • Enforce by Invalid_Type

      • The following policies examine the type of invalidity which will be “INVALID_ASN” or “INVALID_LENGTH”. INVALID_ASN means that the AS announcing the prefix is not the one specified in the ROA. INVALID_LENGTH, means that, although the correct AS is announcing the prefix, the length is outside the range specified in the ROA. This policy returns
      • 2. Enforce-Invalid_ASN-Only
        • Output: Enforce-Invalid_ASN-Only returns:
          • 2 “Enforce” if Invalid_ASN is True,
          • 0 “Pass” otherwise.
        • Input: wherether invalid by asn (boolean invalid_asn).
      • 3. Enforce-Invalid_Length-Only
        • Output: Enforce-Invalid_Length-Only returns:
          • 2 “Enforce” if Invalid_Length is True,
          • 0 “Pass” otherwise.
        • Input: wherether invalid by length (boolean invalid_length).
      • 4. Enforce-Invalid_Both-Only
        • Output: Enforce-Invalid_Length-Only returns:
          • 2 “Enforce” if Invalid_Length is True and Invalid_ASN is True,
          • 0 “Pass” otherwise.
        • Input: wherether invalid by length or invalid by asn (boolean invalid_length, boolean invalid_asn)

    • Pass-If-No-Alternative-Route

      • 5. Version-1

        • Output: Pass-If-No-Alternative-Route-Version-1 returns:
        • 2 “Pass” if No alternative prefix from neighbor ASes to to the origin exists.
        • 1 “Deprefer” otherwise

      • Input: origin of the next best route (integer n)

      • 6. Version-2

        • Output: Pass-If-No-Alternative-Route-Version-2 returns:
        • 2 “Pass” if No alternative prefix from neighbor ASes or less specific prefix in the current as to the origin exists.
        • 1 “Deprefer” otherwise

      • Input:

        • Origin of the next best route (integer n)
        • List of prefix-origin pairs for current ASN (pandas.DataFrame df)
